# Content Templates

Fill in the bracketed fields to generate content. Each template follows the problem-first methodology.

---

## Template 1: The Hidden Cost (适用场景：效率工具、B2B、SaaS)

**Target:** Users who don't realize they're wasting time/money on a daily habit.

### Xiaohongshu Version
```
Title: 你每天[重复动作]的那[时间量]，一年加起来是[震撼数字]

[具体人物]每天要[描述日常工作流]。
我观察了一下，发现ta在[具体低效环节]上，每次要花[具体时间]。
一天[次数]次，就是[日浪费]。一个月就是[月浪费]。

ta不是[常见归因，如"效率低"]，是[真正原因]。

后来ta改了一个步骤：[解决方案，产品自然出现在流程中]。
现在每天多出[节省时间]，拿来[更有价值的事]。
```

### Douyin Version
```
[0-3s] "我录了一下[某人]的屏幕——[时间段]内，ta[具体低效动作][次数]次。"
[3-15s] "每次[动作]平均打断[秒数]秒。[次数]次就是[总时间]。
         一年[工作日]个工作日，就是整整[天数]天。"
[15-30s] "ta不是[错误归因]，是[工作流]里有一个[形象比喻]——[真正问题]。"
[30-45s] [录屏展示新流程，产品作为流程的一部分出现]
         "以前[旧时间]的事情，现在[新时间]。"
[45-55s] 自然收尾，不推销
```

## Template 2: The Wrong Method (适用场景：教育、技能培训、内容工具)

**Target:** Users who are doing something the wrong way and getting bad results.

### Xiaohongshu Version
```
Title: 你[做某事]的方式，可能正在让[负面结果]越来越严重

[人物]用[常见方法]做了[具体事情]，结果[量化的坏结果]。
ta改了[次数]遍，[结果更差的量化数据]。

不是[工具/方法]不好，是ta[使用方式]就是错的。

大多数人[做这件事]的方式是：[描述错误流程]。
但[解释为什么这个流程注定失败]。
你以为[用户的认知]，其实[真相]。

正确的做法是[描述正确流程，产品自然嵌入]。
[量化的好结果]。
```

### Douyin Version
```
[0-3s] "[人物]用[方法]做了[事情]，结果[坏结果]。
        ta改了[次数]遍，反而[更差]。不是[方法]不行——"
[3-15s] "是ta[使用方式]本身就是错的。[解释真正原因]。"
[15-35s] "正确的做法是[步骤]"
         [录屏展示正确流程]
[35-50s] "[结果对比]。"
[50-60s] "有人问我用的什么工具——[一句话带过，不展开]。"
```

## Template 3: The Counterintuitive Truth (适用场景：任何品类的涨粉爆款)

**Target:** Anyone who holds a common belief that's actually wrong.

### Xiaohongshu Version
```
Title: 为什么你[做了正确的事]之后，[反面结果]反而更严重了？

[人物]跟我说，自从[做了某事]，ta的[指标]反而[变差了]。
我说怎么可能？ta说——[解释现象]。

这是[领域]的一个隐藏陷阱：
[你以为的因果关系]，其实[真正的因果关系]。

问题不在于[表面原因]，而在于[深层原因]。

ta后来做了一件事：[描述转变，产品作为转变的一部分]。
[时间]后，[量化的正面结果]。
```

## Template 4: The Comparison Reveal (适用场景：竞品对比、模型对比、方案选择)

**Target:** Users struggling to choose between options.

### Xiaohongshu Version
```
Title: [选项A]和[选项B]到底该用哪个？花了[金额]帮你测了

先说结论：不存在"哪个更好"，只有"哪个适合你的场景"。

我拿同一个[任务]分别让[A]和[B]做，发现：
- [场景1]：[A]明显更好，因为[具体原因]
- [场景2]：[B]赢了，因为[具体原因]
- [场景3]：差不多，但[细微差别]

关键不是选哪个，是[真正该关注的问题]。

[一句话带过自己如何同时使用多个选项的方式]
```

### Douyin Version
```
[0-3s] "我让[数量]个AI互相PK，结果最强的竟然是——" [暂停/悬念]
[3-25s] [分屏展示同一任务的不同输出，标注差异]
[25-40s] "所以答案是：看场景。[场景A]用[A]，[场景B]用[B]。"
[40-50s] "想同时用的话——[一句话描述使用方式，产品自然出现]"
[50-60s] "你们平时用哪个？评论区聊聊"
```

## Template 5: The Client Story (适用场景：服务类、B2B、代运营)

**Target:** Potential clients who need social proof through narrative.

### Xiaohongshu Version
```
Title: 我帮[身份]做了[具体事情]，[量化结果]

[身份描述]找到我，说ta的[问题描述]。
我看了一眼就知道问题在哪——

ta的[内容/产品/流程]在说"[ta以为该说的]"，
而不是"[用户真正需要听到的]"。

和我们[做内容/做产品]一个道理：
[用户看到你的东西时，ta还不觉得自己有问题]。
你上来就说"[功能描述]"，ta关掉了。
你应该先说"[用户能产生共鸣的问题]"。

我帮ta[具体做了什么，工具/方法自然出现]。
[时间]后，[量化的改善结果]。
```

## Segment-Specific Angles

Quick reference for adapting the above templates to different user segments:

| User Segment | Hidden Problem Angle | Scenario Setting |
|---|---|---|
| Developers | "The time you spend setting up tools eats the time the tools save you" | Late night debugging, environment config hell |
| Students | "Your study method is actually making retention worse" | Exam prep, thesis writing, language learning |
| Freelancers | "You're undercharging because you're spending time on the wrong tasks" | Client work, proposal writing, invoicing |
| Small business | "Your 3-person team is doing 10 people's work — but in the wrong 10 things" | Operations, customer service, marketing |
| General users | "The app you use every day has a feature you've never clicked" | Daily workflow, phone habits, content consumption |
